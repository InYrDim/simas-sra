Blocked by: 

## Question
How should the Admin Form Builder UI look and behave? Specifically, how do we implement the simple field list (add/remove/edit), setting required/optional, and the "apply template" action (which clears existing fields and appends the template)? Let's build a prototype to test this UX.

## Answer
Variant A (Stacked Cards) was chosen by the user. Fields are displayed as stacked cards with inline controls for editing labels, types, and the required toggle. The "Apply Template" button is placed at the top header, which clears the existing form state and appends the template fields.
